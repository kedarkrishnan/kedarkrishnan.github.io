    Object oriented programming concepts revisited (Java)

    Encapsulation

    Keeping unnecessary details hidden.

    Encapsulation is something like creating a black box for the end user. We as a user can operate the LED TV and watch our desired shows on it, but if the TV stops working suddenly we would do a basic check as for the power and the cable lines connection but beyond that the TV is a black box for us. A TV mechanic can further test the components of the TV and if the fault is inside a particular component for him that component is a black box and he would ask us to get it replaced or fixed from the company.

    In Software also hidding the unnecessary details helps in problem solving. The end user does not have to worry about the details of how the component works but can concentrate on his main business.

    Abstraction

    Keeping things relevant.

    Why does the TV not include components or circuit for a control of a dish washer or washing machine? Abstraction is all about removing unwanted functionalities till only the ones relavant to the solution remains.

    Polymorphism

    Varying behaviour.

    Reusability is an important aspect in software development. Polymorphism is the key mechanism to achive to achive this in Object Oriented Programming.

    Using Interface

    Suppose we need to write a class which would provide the sum and average of values passed to it. So if the class is provided with number of recantages it should provide the average and sum of areas of all rectangles but if bank accounts are passed it should provide sum and average of balances of all bank account. Thus we need a common code which would work on different objects.

    This can be done by creating an interface say 'Measurable' which both the 'BankAccount' and 'Rectangle' would implement and provide what needs to be measured in its 'getMeasure' method. You can download the sample code from here.

    The selection of correct 'getMeasure' method is not done at compilation time the virtual machine selects the corret method based on the object. This method of selection is called late binding.

    Inheritance

    Is-a relationship.

    Inheritance is a technique of enhancing an existing class. If you need to implement a new class which is more specific to a general concept class then the new class can inherit from the existing class. Example: If we need to define a class 'SavingAccount' to model an account that pays fix interest rate and an a 'FixedDepositAccount' to model an account that pays higher interest rate but the money is locked in for a period then both the classes can inherit from a more genral model 'BankAccount' which can abstract the common code. You can download the sample code from here.

    Inheritance is a relationship between a more general class and a more specific class. This relationship is described as is-a relationship like Saving Account is a Bank Account.


    Another relationship which one should understand is "Has a relationship", it denotes that object of one class has a reference to object of another class.

    Aggregation
    Can exist independently

    Implies a relationship where the child can exist independently of the parent. Example: Study-Class (parent) and Student (child). Delete the Study-Class and the Students still exist.

    Composition
    Cannot exist independently

    Implies a relationship where the child cannot exist independent of the parent. Example: Hand (parent) and Fingers (child). Fingers don't exist separate to a Hand.

    Dependency
    Changing one may affect the other

    Relates to collaboration or delegation, where an object requests services from another object and is therefor dependent on that object.

    To summarize the key concepts of object oriented programming are:

    • Encapsulation - Keeping unnecessary details hidden
    • Abstraction - Keeping things relevant
    • Polymorphism - Varying behaviour
    • Inheritance - Is-a relationship
